# Open Analytics AI

**Natural Language to SQL Analytics Platform**  
100% Open Source · Self-Hosted · Runs Completely Offline

---

## What It Does

Upload an Excel or CSV file, ask a question in plain English, and instantly get:
- SQL generated by a local LLM (Ollama)
- Interactive data table with sorting, filtering, pagination
- Auto-generated charts (line, bar, pie, scatter, heatmap)
- KPI cards
- Export in CSV, Excel, PDF, HTML, PowerPoint

> "Show me serious adverse events by drug" → SQL → Dashboard

## Quick Start

### Prerequisites
- Docker & Docker Compose
- 8+ GB RAM (16 GB recommended for larger models)
- ~10 GB disk space for models

### 1. Clone & Start
```bash
git clone https://github.com/your-org/open-analytics-ai
cd open-analytics-ai
cp .env.example .env
docker compose up -d
```

### 2. Pull an LLM Model
```bash
docker exec analytics-ollama ollama pull llama3.1
# Or smaller/faster models:
# docker exec analytics-ollama ollama pull qwen2.5:7b
# docker exec analytics-ollama ollama pull phi4
# docker exec analytics-ollama ollama pull mistral
```

### 3. Open the App
- **App**: http://localhost:80
- **API Docs**: http://localhost:8000/api/docs
- **Default login**: `admin` / `admin123`

## Features

### Data Ingestion
- Multi-file Excel upload (all sheets automatically ingested)
- CSV upload with auto-detection
- Drag & drop interface
- Up to 500 MB per file
- Automatic schema detection and column typing
- Relationship discovery between tables
- Data quality scoring (null %, duplicate %, overall score)

### AI Query
- Natural language → SQL via local Ollama LLM
- Schema-aware prompting (table names, columns, sample values sent to LLM)
- Conversation history (last 3 exchanges used for context)
- SQL validation before execution
- Auto-suggested questions per dataset
- SQL editor with syntax highlighting and Ctrl+Enter to run
- Query explanation in plain English

### Security
- Only `SELECT`, `WITH`, `GROUP BY`, `ORDER BY`, `HAVING`, `LIMIT` allowed
- Pattern matching blocks: DELETE, UPDATE, INSERT, DROP, ALTER, MERGE, TRUNCATE, EXECUTE
- SQL injection prevention (null bytes, comment stripping, hex literals)
- Automatic LIMIT injection (defaults to 1,000 rows, max 100,000)
- JWT authentication
- Role-based access (admin, analyst, viewer)

### Visualization
- Auto-selected chart types based on data shape:
  - Line chart: date/time + numeric columns
  - Bar chart: categorical + numeric (top 15)
  - Pie chart: 2–8 unique categories
  - Scatter: 2+ numeric columns
  - Correlation heatmap: 3+ numeric columns
- KPI cards: sum, average, count per numeric column

### Export
| Format | Contents |
|---|---|
| CSV | Raw data |
| Excel | Styled table + metadata sheet |
| PDF | Formatted table (ReportLab) |
| HTML | Self-contained styled page |
| PowerPoint | Title slide + data table + stats slide |

## Supported Models (via Ollama)

| Model | Size | Notes |
|---|---|---|
| `llama3.1` | 8B (4.7 GB) | Recommended default |
| `qwen2.5:7b` | 7B (4.4 GB) | Excellent SQL quality |
| `deepseek-r1:7b` | 7B | Strong reasoning |
| `mistral` | 7B (4.1 GB) | Fast, reliable |
| `phi4` | 14B (9 GB) | High quality, needs more RAM |
| `gemma2:9b` | 9B (5.4 GB) | Google's model |

Change model in the top bar dropdown at any time.

## Configuration

Edit `.env`:
```
SECRET_KEY=your-random-secret-here
DEBUG=false
OLLAMA_BASE_URL=http://ollama:11434
DEFAULT_MODEL=llama3.1
MAX_FILE_SIZE_MB=500
QUERY_TIMEOUT=30
```
